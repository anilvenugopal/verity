# Schema assembly & verification report

**Date:** 2026-05-31

## Load order

The seven fragments are NOT independent schemas ÔÇö they are authoring fragments of ONE database spanning four PostgreSQL schemas (`governance`, `runtime`, `compliance`, `analytics`). Concatenating them verbatim fails: duplicate `CREATE TYPE`/`CREATE TABLE`, conflicting enum member sets, and forward FK references. Assemble in dependency order, applying the de-duplication patches in the next section as you go.

### SQL header (top of verity_schema.sql)

```sql
-- =====================================================================
-- verity_schema.sql ÔÇö Verity v2 canonical schema (ADR-0005 hardened)
-- Generated by assembling per-domain hardened fragments. Single owner per
-- object; see "Cross-domain reconciliation" for the de-dup patches applied.
-- Target: PostgreSQL 18+ (uuidv7() builtin, mandated generator per
-- naming-conventions.md ┬ž3).
-- =====================================================================

-- 1. Extensions (must precede all type/table DDL).
CREATE EXTENSION IF NOT EXISTS pgcrypto;   -- gen_random_uuid() (PG<18 fallback only)
CREATE EXTENSION IF NOT EXISTS vector;      -- pgvector: agent/task/tool/prompt/requirement/mart_field embeddings

-- 2. Namespaces.
CREATE SCHEMA IF NOT EXISTS governance;
CREATE SCHEMA IF NOT EXISTS runtime;
CREATE SCHEMA IF NOT EXISTS compliance;
CREATE SCHEMA IF NOT EXISTS analytics;

-- 3. search_path: governance first (owns the uuidv7() fallback shim + most types),
--    then runtime/compliance/analytics, then public.
SET search_path = governance, runtime, compliance, analytics, public;

-- 4. UUIDv7 generator. PG18+ provides uuidv7() builtin (resolved via public).
--    On PG<18 ONLY, install ONE shim, unqualified-resolvable, BEFORE any table:
-- CREATE FUNCTION public.uuidv7() RETURNS uuid LANGUAGE sql VOLATILE
--   AS $$ SELECT gen_random_uuid() $$;  -- replace with a true v7 impl in prod.
--    Every fragment calls bare uuidv7(); the `decisions` fragment's
--    governance.uuidv7() calls MUST be rewritten to bare uuidv7() (see reconciliation #E1).
```

## Cross-domain reconciliation

1. **`governance.lifecycle_state`** ÔÇö declared in `entities`, `lifecycle_approvals`, `packages_deploy`. Keep the `entities` definition only. Delete the `CREATE TYPE governance.lifecycle_state ...` blocks from `lifecycle_approvals` and `packages_deploy` (members are identical; safe).

NOTE: `materiality_tier` is in BOTH `entities` and `intake` with identical members ÔÇö pick `intake` as owner and delete from `entities` (or vice versa; pick one, delete the other).
4. **`governance.ai_risk_tier`** ÔÇö keep `intake`; delete from `lifecycle_approvals`.
5. **`governance.approval_request_status`** ÔÇö defined in both `intake` (`pending/approved/rejected/cancelled`) and `lifecycle_approvals` (`pending/approved/rejected/withdrawn`). MEMBER CONFLICT. 6. **`governance.run_purpose`, `decision_log_detail`, `entity_type`** ÔÇö keep `entities`; delete the `decisions` re-declarations. 8. **`deployment_action`/`deployment_operation`, `deployment_outcome`, `deployment_run_mode`** ÔÇö three conflicting definitions each.
