# Schema assembly & verification report

**Date:** 2026-05-31

## Load order

The seven fragments are NOT independent schemas — they are authoring fragments of ONE database spanning four PostgreSQL schemas (`governance`, `runtime`, `compliance`, `analytics`). Concatenating them verbatim fails: duplicate `CREATE TYPE`/`CREATE TABLE`, conflicting enum member sets, and forward FK references. Assemble in dependency order, applying the de-duplication patches in the next section as you go.

### SQL header (top of verity_schema.sql)

```sql
-- =====================================================================
-- verity_schema.sql — Verity v2 canonical schema (ADR-0005 hardened)
-- Generated by assembling per-domain hardened fragments. Single owner per
-- object; see "Cross-domain reconciliation" for the de-dup patches applied.
-- Target: PostgreSQL 18+ (uuidv7() builtin, mandated generator per
-- naming-conventions.md §3).
-- =====================================================================

-- 1. Extensions (must precede all type/table DDL).
CREATE EXTENSION IF NOT EXISTS pgcrypto;   -- gen_random_uuid() (PG<18 fallback only)
CREATE EXTENSION IF NOT EXISTS vector;      -- pgvector: agent/task/tool/prompt/requirement/mart_field embeddings

-- 2. Namespaces.
CREATE SCHEMA IF NOT EXISTS governance;
CREATE SCHEMA IF NOT EXISTS runtime;
CREATE SCHEMA IF NOT EXISTS compliance;
CREATE SCHEMA IF NOT EXISTS analytics;

-- 3. search_path: governance first (owns the uuidv7() fallback shim + most types),
--    then runtime/compliance/analytics, then public.
SET search_path = governance, runtime, compliance, analytics, public;

-- 4. UUIDv7 generator. PG18+ provides uuidv7() builtin (resolved via public).
--    On PG<18 ONLY, install ONE shim, unqualified-resolvable, BEFORE any table:
-- CREATE FUNCTION public.uuidv7() RETURNS uuid LANGUAGE sql VOLATILE
--   AS $$ SELECT gen_random_uuid() $$;  -- replace with a true v7 impl in prod.
--    Every fragment calls bare uuidv7(); the `decisions` fragment's
--    governance.uuidv7() calls MUST be rewritten to bare uuidv7() (see reconciliation #E1).
```

## Cross-domain reconciliation

These are fixes no single per-domain agent could see because they require the union. Apply as exact patches.

### A. Collapse duplicated shared enums to one owner (load-blocking)

1. **`governance.lifecycle_state`** — declared in `entities`, `lifecycle_approvals`, `packages_deploy`. Keep the `entities` definition only. Delete the `CREATE TYPE governance.lifecycle_state ...` blocks from `lifecycle_approvals` and `packages_deploy` (members are identical; safe).
2. **`governance.deployment_channel`** — keep `entities`; delete from `lifecycle_approvals`.
3. **`governance.materiality_tier`, `approval_role`, `approval_decision`, `approval_request_kind`** — keep `intake`; delete from `lifecycle_approvals` and (for `materiality_tier`) from `entities`. NOTE: `materiality_tier` is in BOTH `entities` and `intake` with identical members — pick `intake` as owner and delete from `entities` (or vice versa; pick one, delete the other).
4. **`governance.ai_risk_tier`** — keep `intake`; delete from `lifecycle_approvals`.
5. **`governance.approval_request_status`** — defined in both `intake` (`pending/approved/rejected/cancelled`) and `lifecycle_approvals` (`pending/approved/rejected/withdrawn`). MEMBER CONFLICT. Reconcile to one set before delete. Recommend `intake`'s `cancelled` OR add both (`pending/approved/rejected/cancelled/withdrawn`); keep one owner (intake), delete the other.
6. **`governance.run_purpose`, `decision_log_detail`, `entity_type`** — keep `entities`; delete the `decisions` re-declarations. MEMBER CONFLICT on `decision_log_detail`: `entities` = `minimal/standard/verbose`, `decisions` = `minimal/standard/full`. Pick one before deleting. v1 inventory shows only free-text `'standard'`; recommend keeping `entities`' `minimal/standard/verbose` and changing `decisions` code to match.
7. **`governance.package_kind`** — declared in `entities`, `lifecycle_approvals`, `packages_deploy` (identical members `vtx/vax`). Keep `packages_deploy`; delete from the other two.
8. **`deployment_action`/`deployment_operation`, `deployment_outcome`, `deployment_run_mode`** — three conflicting definitions each. Canonicalize to `packages_deploy` (most complete): keep `governance.deployment_operation` (`deploy_nonprod, deploy_prod, promote_champion, lock_deprecated, cleanup_deprecated, rollback`), `governance.deployment_outcome` (`requested, rejected_incompatible, rejected_lifecycle, rejected_unauthorized, succeeded, failed, superseded`), `governance.deployment_run_mode` (`live, read_only, ab_slice, locked`). Delete the `entities` and `lifecycle_approvals` enum copies and rewrite their (deleted) tables' references — see B.

## Verification report

Prioritized review checklist for the human reviewer (ADR-0005 reviewed-artifact gate). The schema is NOT acceptable until every Critical item is resolved and every Should-fix item is dispositioned (fixed or explicitly waived with reason).

### Critical (load-blocking or correctness; MUST fix before acceptance)

- [ ] **C1. De-duplicate shared enums to one owner** (reconciliation A1–A8). The verbatim union throws `42710 duplicate_object`. Verify each shared enum is `CREATE`d exactly once after assembly: `lifecycle_state, deployment_channel, materiality_tier, ai_risk_tier, approval_role, approval_decision, approval_request_kind, approval_request_status, package_kind, run_purpose, decision_log_detail, entity_type, deployment_outcome, deployment_run_mode, deployment_action/operation`.
- [ ] **C2. Reconcile CONFLICTING enum member sets** before deleting duplicates: `decision_log_detail` (`verbose` vs `full`), `approval_request_status` (`cancelled` vs `withdrawn`), `deployment_outcome` (3-way), `deployment_run_mode` (`ab_slice` vs `ab`; `locked` presence). A wrong pick silently changes the spec. Confirm the chosen set matches v1 intent + ADR-0006.
- [ ] **C3. De-duplicate shared tables to one owner** (reconciliation B9–B15): identity table, `application`, `model`/`model_price`, `approval_request`/`approval_signoff`, the three `intake_*` estimate/roi/envelope tables, `harness_image`/`package`/`package_*compat`/`deployment`, `run_dispatch_outbox`. Verify no `42P07 duplicate_table` and no two definitions with divergent columns survive.
- [ ] **C4. Resolve `account_user` vs `app_user`** (reconciliation B9). naming-conventions.md §1 uses `account_user` verbatim; `auth` fragment uses `app_user`. Pick one; verify all FK columns, grant tables, event tables, and current-role views use it consistently. This blocks every downstream identity FK (C19, E2–E4).
- [ ] **C5. `embedding_config` schema mismatch** (reconciliation C20): `intake` FKs `governance.embedding_config`; `reporting` defines `compliance.embedding_config`. The `intake_requirement` FK will fail to resolve. Decide the owning schema and fix both the definition and the FK.
- [ ] **C6. Drop mutable champion pointers** (reconciliation D21). `agent.current_champion_version_id`/`task.current_champion_version_id` contradict the append-only `champion_assignment` source of truth. Two writable sources of "current champion" is a correctness defect.
- [ ] **C7. Tautological CHECK enforces nothing.** `compliance.report_definition.ck_report_definition_template_requires_docx` contains `... OR EXISTS (SELECT 1)`, which is always true (and a CHECK cannot reference another table). The "template_driven requires docx or sql_template" invariant is UNENFORCED. Remove the tautology; enforce template/sql presence in the API or via the 1:1 `report_sql_template`.
- [ ] **C8. `auth_event` June partition missing.** The surviving `auth` fragment ships only `auth_event_2026_05`. Today is in 2026-05; a 2026-06 insert fails (no partition). Add `auth_event_2026_06` and document the monthly partition-creation job. (All other Tier-2 tables ship `_05` and `_06`.)
- [ ] **C9. NO-SILENT-CAPABILITY-LOSS — entire GT/test/validation subsystem unmapped.** 4 enums (`gt_dataset_status`, `gt_quality_tier`, `gt_source_type`, `gt_annotator_type`) and 17 tables (`test_suite`, `test_case`, `test_case_mock`, `test_execution_log`, `ground_truth_dataset`, `ground_truth_record`, `ground_truth_record_mock`, `ground_truth_annotation`, `validation_run`, `validation_record_result`, `evaluation_run`, `model_card`, `metric_threshold`, `field_extraction_config`, `incident`, `description_similarity_log`, `platform_settings`) have NO hardened object AND NO mapping disposition. - [ ] **C10. Dependency break: deferred view has no source.** `analytics.v_validation_result` (DEFER'd) reads `governance.test_execution_log`, which is unmapped/dropped (C9). The deferred rebuild has no source table — resolve as part of C9.
