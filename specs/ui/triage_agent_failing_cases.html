Root cause — prompt regression in v1.5.0
The exclusions line in triage_system_prompt v1.5.0 was changed from "securities, class actions" to add class actions — but the prior text explicitly included "hazardous materials run-off" as a hard RED trigger. That phrase was present in v1.4.0 and was silently dropped when the exclusion line was restructured. Restore the hazardous materials run-off rule in the exclusions list. The corrected line should read:

This is a prompt-only fix — no agent version wiring change required. Create triage_system_prompt v1.5.1, wire it into triage_agent v2.2.1, and re-run the full test suite before promoting to candidate. Also add a targeted adversarial case for run-off policies with thin financials to prevent regression.

Root cause — new exclusion created ambiguity, not clarity
This failure is the inverse of case 1. The addition of "class actions" to the exclusions list in v1.5.0 was intended to flag submissions where the named insured faces active class action exposure. However, Meridian's prior claim was a class action that has already settled — it is historical, not active. The prompt does not distinguish between "currently subject to class action" and "previously settled a class action." The model, seeing "class actions" in the exclusions block and "class_action" in the prior claim context, concludes this is an AMBER flag rather than a RED disqualifier — because it cannot determine whether the rule applies to historical or current exposure.

The baseline (RED) was set when v1.5.0 was the candidate — the annotator's expectation was RED because the loss ratio of 0.74 combined with a 2-year-old class action settlement in investment advisory is a hard decline by appetite policy. The model at 0.83 confidence is not uncertain — it has formed a considered view — but that view is wrong because the exclusion rule is underspecified.
Blame: triage_system_prompt v1.5.0 · authored by J. Morris · 2026-05-28 · "Add class actions to exclusions list" same commit as case 1 · two failure modes from one prompt change
Remediation
The exclusion needs temporal precision. The corrected instruction should distinguish active from historical exposure:
- Exclusions: sanctions, securities, active class actions, hazardous materials run-off
- Flag (not exclude): prior settled class actions within 5 years with loss ratio > 0.65

This separates the hard-RED exclusion (active class action) from the AMBER-then-senior-escalation path (historical settlement + adverse loss ratio). The second rule should be added as a standalone instruction block, not appended to the exclusions list — mixing rules and flags in a single list is what caused the ambiguity in the first place. Also update the inference config: the model should be instructed to reason about temporal context before applying exclusion rules.
Summary — both failures trace to the same commit

Both cases should be added to the adversarial suite before v2.2.1 is promoted to staging.
