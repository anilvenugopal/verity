Navigation philosophydesign-system §7 · apps-based model

Three navigation zones, each answering one question, nothing reachable two ways. The left rail switches apps; the sidebar switches sections within an app; the top bar tells you where you are (breadcrumb) and what you are scoped to (Application Scope), plus global tools. The top bar is deliberately not an app switcher — that would duplicate the rail.

Composition · Tests · Performance
1
Left rail → “Which app?”
Icon-only, global, always present. The single switcher between apps (Home, Studio, Registry…). The brand logo is not here — it lives top-left of the bar.
2
Sidebar → “Which section of this app?”
Labeled, in-app. Sections of the current app only (Studio → Agents / Tasks / Prompts). Collapses to rail-only below 1280px.
3
Top-bar breadcrumb → “Where am I?”
Context › App › Section › Entity. When deep, the middle collapses to first › … › last; the … opens every level. Logo sits to its left.
4
Application Scope → “Which application(s)?”
A global filter bounding every page to all applications or a selected set. Set anytime; locked to the page’s application on Studio and app-scoped detail screens.
The rule.One question per zone; one path per destination. If a place is reachable from two zones, one of them is doing the wrong job.
Application Scope.A portal-wide filter, not a location. Selecting application(s) constrains the data shown on every page at once; it is disabled and locked to the current application inside Studio and app-scoped detail screens.
