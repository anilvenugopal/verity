Application Onboarding

Register an AI application with the governance platform before adding use cases.

Purpose

An application is the organisational container for AI use cases. It declares the regulatory frameworks, jurisdictions, and data classification ceiling that apply to all intakes under it. An application must be approved before any use case (intake) can be created.

Fields

Data classification ceiling
The highest sensitivity tier any intake under this application may reach. Constrains downstream use cases.
Regulatory frameworks
The compliance frameworks this application is subject to (e.g. EU AI Act, NAIC Model Bulletin).
Business owner
Accountable for governance compliance. Receives approval requests.

What happens next

After submission, AI Governance reviews the onboarding request. Once approved, the application becomes active and use cases (intakes) can be created under it.
