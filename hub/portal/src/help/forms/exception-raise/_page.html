Raise Exception

Request a time-limited waiver for a compliance obligation that cannot be fully satisfied.

Purpose

An exception acknowledges that a compliance obligation exists but cannot be met in full. It documents the compensating controls, the rationale, and an expiry date after which the exception must be reviewed. Exceptions require approval by a compliance or security approver who did not raise the request (separation of duty).

Fields

Compensating controls
What is in place to mitigate the risk. Required — the approver evaluates the adequacy of these controls.
Rationale
Why full satisfaction is not achievable right now.
Expiry date
Exceptions are time-limited. Set a realistic review date, not a distant one.
