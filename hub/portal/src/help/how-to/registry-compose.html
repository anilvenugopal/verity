Composing an Agent Version

Assign prompts, tools, and MCP servers to an agent version.

Prompt assignments

Open the agent version detail page and expand Prompt assignments. Click + Assign prompt, paste the prompt version ID, choose the API role (system, user, or assistant_prefill), and submit. An agent version must have at least one prompt assignment before it can be promoted to champion.

To remove an assignment, click Remove on the row. Removed assignments do not affect other versions that reference the same prompt version.

Tool assignments (agent-only)

Tools can only be assigned to agent versions, not tasks. Click + Assign tool, select the tool version, and submit. Attempting to assign a tool to a task version returns a 409 error.

Source and target bindings

Bindings describe data sources (documents, structured inputs) and targets (write destinations) the agent reads from or writes to. Storage-object bindings require a data connector version. Add them via the API or YAML import.

YAML bundle export / import

Use GET /api/registry/versions/{id}/export to download a version's full composition as a YAML bundle. Use POST /api/registry/import/dry-run with the YAML body to verify it is already registered (all entries show no_op). The import endpoint is idempotent by natural key.
