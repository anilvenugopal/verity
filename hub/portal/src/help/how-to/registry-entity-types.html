Registry Entity Types

Agents, tasks, prompts, tools, and models — what they are and how they relate.

Agents

An agent is an autonomous AI executable — it has a prompt composition, optional tool and MCP server assignments, and an inference configuration. Agents are the primary governed asset; they go through the full lifecycle (draft → champion) and are subject to intake approval before reaching production stages.

Tasks

A task is a structured subtask executed by or on behalf of an agent. Tasks share the lifecycle and prompt-assignment machinery but do not have tool or MCP assignments (those are agent-only).

Prompts

A prompt is a versioned, content-hashed template assigned to agent or task versions via a role slot (system, user, or assistant_prefill). Multiple prompt versions can coexist; each assignment pins a specific version.

Tools

A tool is a versioned external capability (function call, API endpoint). Tools carry a transport code and an optional data classification. Only agents can have tool assignments — tasks cannot.

Models

A model is an LLM entry in the catalog (e.g. claude-3-haiku). Models have SCD-2 pricing history and are referenced indirectly via model references — symbolic aliases (e.g. "primary-llm") that can be rebound to a different model without changing the agent configuration.
